    Libraries

    The Lua environment provided by rbxmk is packaged as a list of libraries.

    In rbxmk, a library is most accurately described as a set of modifications to the Lua environment. Some libraries group functions under a single table as normally expected, while others add a number of functions directly to the global table. Some libraries add a number of tables, and others merge with tables that already exist. Libraries are loaded in ascending order by their priority value.

    The following libraries are available:

    LibraryPriorityDescription
    base-1The Lua 5.1 standard library, abridged.
    program0Values related to the running program.
    rbxmk1An interface to the rbxmk engine, and the rbxmk environment.
    roblox2An environment emulating the Roblox Lua API.
    clipboard10An interface to the clipboard.
    fs10An interface to the file system.
    http10An interface to network resources.
    math10Extensions to the standard math library.
    os10Extensions to the standard os library.
    path10File path manipulation.
    rbxassetid10An interface to Roblox assets.
    string10Extensions to the standard string library.
    sym10Symbols for accessing instance metadata.
    table10Extensions to the standard table library.
    types10Fallbacks for constructing certain types.

    Filtering #

    The run and interactive commands have a --libraries flag, which allows the user to include and exclude libraries.

    A library can have a number of types as dependencies. When the library is included, it will pull these types into the environment. For example, The roblox library depends on the CFrame type, so including it will also include the CFrame table into the global environment.

    Conversely, if no included libraries depend on a type, then that type will not be included.
